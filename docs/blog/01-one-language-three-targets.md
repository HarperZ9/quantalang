# QuantaLang: One Language, Three Shader Targets

*The shader portability problem is solved. Write once, compile to HLSL, GLSL, and ReShade -- zero rewrites, zero failures.*

---

## The Problem: Writing the Same Shader Three Times

If you ship graphics software that targets multiple APIs, you know the pain. DirectX wants HLSL. OpenGL and Vulkan want GLSL (or SPIR-V). ReShade wants HLSL-flavored `.fx` files with specific conventions. That means every post-processing effect you write needs to exist in at least two or three forms.

The typical workflow looks like this:

1. Write the algorithm in HLSL for DirectX.
2. Port it to GLSL for OpenGL/Vulkan, translating every type name, every intrinsic, every semantic.
3. Wrap it in ReShade boilerplate for the modding community.
4. Maintain all three versions forever.

This is not a minor inconvenience. Consider a vignette shader -- one of the simplest post-processing effects. Even for 20 lines of math, porting introduces subtle differences:

- `float4` vs `vec4`
- `tex2D(sampler, uv)` vs `texture(sampler, uv)`
- `SV_Position` semantics vs `gl_FragCoord`
- ReShade's `#include "ReShade.fxh"` and `technique` blocks

Multiply this by every effect in your pipeline. Multiply it again by every bug fix that has to land in three places simultaneously. It does not scale.

---

## The Solution: QuantaLang

QuantaLang is "The Effects Language." You write shader logic once in `.quanta` syntax and the compiler emits clean, idiomatic output for each target.

Here is the core idea in one command:

```bash
# Same source, three targets
quantac vignette.quanta --target hlsl -o vignette.hlsl
quantac vignette.quanta --target glsl -o vignette.glsl
quantac vignette.quanta --target hlsl -o vignette.fx    # ReShade .fx
```

No preprocessor hacks. No `#ifdef GL_ES`. No template metaprogramming. The compiler understands the semantics of each target and generates correct code for all of them.

## Side-by-Side: One Vignette, Three Outputs

### QuantaLang Source

```rust
fn vignette(uv_x: f64, uv_y: f64, strength: f64, softness: f64) -> f64 {
    let dx = uv_x - 0.5;
    let dy = uv_y - 0.5;
    let dist = sqrt(dx * dx + dy * dy);
    let vig = smoothstep(0.5, 0.5 * softness, dist);
    1.0 - strength * (1.0 - vig)
}

#[fragment]
fn PS_Vignette(uv: vec2) -> vec4 {
    let color = tex2d(uv);
    let vig = vignette(color.x, color.y, 0.5, 0.6);
    vec4(color.x * vig, color.y * vig, color.z * vig, 1.0)
}
```

This is the single source of truth. The `#[fragment]` attribute tells the compiler this function is a pixel shader entry point. `tex2d()` and `vec4` are shader builtins that the compiler knows how to translate per-target.

### HLSL Output (DirectX)

```hlsl
// Generated by QuantaLang Compiler
// Target: HLSL (DirectX / ReShade)

float vignette(float uv_x, float uv_y, float strength, float softness) {
    float dx = (uv_x - 0.5);
    float dy = (uv_y - 0.5);
    float dist = sqrt(((dx * dx) + (dy * dy)));
    float vig = smoothstep(0.5, (0.5 * softness), dist);
    return (1.0 - (strength * (1.0 - vig)));
}

float4 PS_Vignette(float4 pos : SV_Position, float2 uv : TEXCOORD) : SV_Target0 {
    float4 color = tex2D(ReShade::BackBuffer, uv);
    float vig = vignette(color.x, color.y, 0.5, 0.6);
    return float4((color.x * vig), (color.y * vig), (color.z * vig), 1.0);
}
```

Note the automatic translation:
- `f64` becomes `float`
- `vec4` becomes `float4`
- `tex2d()` becomes `tex2D(ReShade::BackBuffer, uv)`
- The `#[fragment]` function gets `SV_Position`, `TEXCOORD`, and `SV_Target0` semantics

### GLSL Output (OpenGL / Vulkan)

```glsl
// Generated by QuantaLang Compiler
// Target: GLSL (OpenGL / Vulkan)

#version 450

double vignette(double uv_x, double uv_y, double strength, double softness) {
    double dx = (uv_x - 0.5);
    double dy = (uv_y - 0.5);
    double dist = sqrt(((dx * dx) + (dy * dy)));
    double vig = smoothstep(0.5, (0.5 * softness), dist);
    return (1.0 - (strength * (1.0 - vig)));
}

vec4 PS_Vignette(vec2 uv) {
    vec4 color = texture(backbuffer, uv);
    double vig = vignette(color.x, color.y, 0.5, 0.6);
    return vec4((color.x * vig), (color.y * vig), (color.z * vig), 1.0);
}
```

Key differences the compiler handles automatically:
- `#version 450` directive
- `tex2d()` becomes `texture(backbuffer, uv)`
- GLSL uses `vec4` natively (no type renaming needed)
- No DirectX-style semantics -- GLSL does not use them

### ReShade .fx Output

```hlsl
// Generated by QuantaLang Compiler
// Target: ReShade Effect (.fx)

#include "ReShade.fxh"

float vignette(float uv_x, float uv_y, float strength, float softness) {
    float dx = (uv_x - 0.5);
    float dy = (uv_y - 0.5);
    float dist = sqrt(((dx * dx) + (dy * dy)));
    float vig = smoothstep(0.5, (0.5 * softness), dist);
    return (1.0 - (strength * (1.0 - vig)));
}

float4 PS_Vignette(float4 pos : SV_Position, float2 uv : TEXCOORD) : SV_Target0 {
    float4 color = tex2D(ReShade::BackBuffer, uv);
    float vig = vignette(color.x, color.y, 0.5, 0.6);
    return float4((color.x * vig), (color.y * vig), (color.z * vig), 1.0);
}

technique Quanta_PS_Vignette {
    pass {
        VertexShader = PostProcessVS;
        PixelShader = PS_Vignette;
    }
}
```

The ReShade target adds:
- `#include "ReShade.fxh"` for the ReShade framework
- A `technique` block binding the pixel shader to the rendering pipeline
- `PostProcessVS` as the default vertex shader (standard ReShade convention)

## How Cross-Target Compilation Works

The QuantaLang compiler pipeline processes shader code through several stages:

```
Source (.quanta)
    |
    v
  Lexer -> Token stream
    |
    v
  Parser -> AST
    |
    v
  Type Checker -> Typed AST
    |
    v
  Target Backend -> Output
    |--- C Backend     -> .c file (CPU execution)
    |--- HLSL Backend  -> .hlsl / .fx file (DirectX / ReShade)
    |--- GLSL Backend  -> .glsl file (OpenGL / Vulkan)
    |--- SPIR-V Backend -> .spv file (Vulkan binary)
```

The key insight is that the frontend (lexer, parser, type checker) is shared. Only the final code generation step is target-specific. This means:

- A type error caught in one target is caught in all targets.
- Optimization passes in the typed AST benefit every backend.
- Adding a new backend does not require changing any shader source.

## What This Means for You

**If you are a game modder:** Write your ReShade effects in QuantaLang. If a game switches from DirectX to Vulkan, recompile -- do not rewrite.

**If you are a graphics developer:** Prototype in QuantaLang, compile to whatever API your engine uses. Ship one set of shader sources in your repo.

**If you are building tools:** QuantaLang also compiles to C. You can run the same math on the CPU for testing, validation, or offline rendering. The `quantac run` command executes shader logic without a GPU.
