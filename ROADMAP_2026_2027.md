### QuantaLang: Output Optimization (Weeks 1-6, parallel with Aurora)

| Deliverable | Detail |
|-------------|--------|
| **Deep copy propagation** | Extend peephole to handle Call terminator → statement chains. Eliminate 80%+ of `_temp` variables from HLSL/GLSL output. |
| **Dead declaration elimination** | Track which locals are actually referenced in the final output. Only declare used variables. |
| **Expression inlining** | For single-use temps that are simple expressions (field access, constants), inline them directly: `float _8 = color.x;` `float _9 = (_8 * vig);` → `float _9 = (color.x * vig);` |
| **Parenthesis cleanup** | Remove redundant parens: `(_8 * vig)` → `_8 * vig` where precedence is unambiguous. |

**Target output for vignette_shader.quanta:**
```hlsl
float4 PS_Vignette(float4 pos : SV_Position, float2 uv : TEXCOORD) : SV_Target0 {
    float4 color = tex2D(ReShade::BackBuffer, uv);
    float vig = vignette(color.x, color.y, 0.5, 0.6);
    return float4(color.x * vig, color.y * vig, color.z * vig, 1.0);
}
```

**Exit criteria:** Generated HLSL for the vignette shader is <20 lines with no unused variables.

## Q3 2026 (July — September): Killer Demo + Developer Experience

**Theme: Write the shader that makes graphics programmers stop scrolling. Make QuantaLang usable by someone who isn't you.**

### QuantaLang: The Killer Shader (Weeks 1-4)

Write one of these ENTIRELY in QuantaLang, compile to `.fx`, verify it works in ReShade:

| Option | Difficulty | Visual Impact |
|--------|-----------|---------------|
| **Screen-Space Ambient Occlusion (SSAO)** | Medium | High — darkens crevices, adds depth |
| **Gaussian Blur (multi-pass)** | Easy | Medium — foundation for DoF, bloom |
| **Depth of Field** | Medium | Very high — cinematic bokeh |
| **Volumetric Light Shafts** | Hard | Very high — god rays |

**Pick one.** SSAO is the best balance of difficulty and impressiveness.

Requirements for the SSAO shader:
- Sample depth buffer via `tex2d_named("depth", uv)`
- Sample normal buffer (reconstruct from depth if needed)
- Random kernel sampling (hemisphere)
- Bilateral blur pass for denoising
- Blend with scene color

**Exit criteria:** `quantac ssao.quanta --target hlsl -o ssao.fx` produces a working ReShade SSAO effect.

## Q1 2027 (January — March): Type System Power + Ecosystem

**Theme: The compiler catches bugs no other tool can find.**

### QuantaLang: Color Space Enforcement in Shaders (Weeks 1-4)

| Deliverable | Detail |
|-------------|--------|
| **Annotations flow through HLSL/GLSL** | `fn tonemap(c: vec3 with ColorSpace<Linear>) -> vec3 with ColorSpace<sRGB>` — the annotation is preserved in the generated shader comments. |
| **Compile error on mixing** | `let wrong = pbr_shade(my_srgb_color)` where `pbr_shade` expects Linear → compiler error with clear message. |
| **Built-in color space types** | `type LinearRGB = vec3 with ColorSpace<Linear>`, `type sRGBColor = vec3 with ColorSpace<sRGB>` in the stdlib. |
| **Conversion functions** | `srgb_to_linear()` changes the annotation from `sRGB` to `Linear`. Verified at compile time. |

**Exit criteria:** Passing sRGB color to a function expecting Linear produces a clear compiler error.

### QuantaLang: Precision Annotations (Weeks 5-8)

| **HLSL `half` emission** | `vec3 with Precision<Half>` emits `half3` in HLSL, `mediump vec3` in GLSL. |

**Exit criteria:** A 10-operation chain with `Half` precision produces a compiler warning about potential precision loss.
